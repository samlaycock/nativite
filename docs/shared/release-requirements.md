# Release Requirements

> Maps to: `README.md`, `src/index.ts`, `src/native/android/generator.ts`,
> `src/native/ios/pbxproj.ts`

Nativite generates native projects and production web bundles. Store-ready
artifacts are still produced by the platform toolchain, so release requirements
come from Apple, Google Play, Xcode, Android Studio, Gradle, and the CI image
used by the app.

The Nativite CLI requires Bun 1.x. `package.json#engines` intentionally does not
advertise Node.js CLI support because the CLI loads `nativite.config.ts`
directly. Use Bun in local release workflows and CI jobs that run `nativite`
commands.

## 1.0 Support Statement

Nativite 1.0 treats the documented configuration helpers, public package
exports, JavaScript bridge APIs, native chrome APIs, generated project
structure, and platform support matrix as supported public surface. Future
minor releases may add capabilities, platforms, plugin hooks, or generated
files, but documented 1.0 contracts should remain compatible unless a future
major release explicitly changes them.

See [Public API Contract](./public-api-contract.md) for the full stable surface,
experimental surface, and semver policy used for release decisions.

## Apple Upload Requirements

Apple owns App Store Connect upload requirements. Starting April 28, 2026,
[Apple's upcoming requirements](https://developer.apple.com/news/upcoming-requirements/)
state that apps uploaded to App Store Connect must be built with Xcode 26 or
later using an SDK for iOS 26, iPadOS 26, tvOS 26, visionOS 26, or watchOS 26.

For Nativite's built-in Apple targets:

- iOS projects generated by `ios()` default to minimum iOS `17.0`.
- macOS projects generated by `macos()` default to minimum macOS `14.0`.
- Generated Xcode projects are release inputs, not the final signed archive.
- Xcode, signing identities, provisioning profiles, notarization credentials,
  archive settings, and upload tooling remain app or CI responsibilities.

## Android Upload Requirements

Google Play owns target API level policy. The
[Google Play target API level policy](https://support.google.com/googleplay/android-developer/answer/11917020)
requires new apps and app updates to target an Android API level within one year
of the latest major Android release. Existing apps that are not updated must
target an API level within two years of the latest major Android release to
remain available to new users on newer Android versions.

For Nativite's built-in Android target:

- `android()` defaults to `minSdk` `26`.
- `android()` defaults to `targetSdk` `36`.
- Generated Android projects set `compileSdk` from `targetSdk`.
- Apps can provide an explicit integer `targetSdk` when they need to track a
  newer Play policy deadline before Nativite changes its default.
- Android Studio, Android SDK installation, Java, Gradle, signing configs,
  keystores, Play Console upload credentials, and final `.aab` or `.apk`
  packaging remain app or CI responsibilities.
